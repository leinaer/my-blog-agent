# 提示词工程基础

**提示词工程（Prompt Engineering）**是设计和优化输入提示，以引导AI模型生成期望输出的技术和艺术。它是与AI有效沟通的核心技能。

<InfoCard type="info" title="为什么提示词如此重要？">

同一个AI模型，不同的提示词可能产生天壤之别的结果。好的提示词可以让AI输出专业、准确、有价值的内容，而差的提示词只会得到泛泛而谈的废话。

</InfoCard>

## 为什么需要提示词工程？

大型语言模型虽然强大，但它们的输出质量很大程度上取决于输入提示的质量。好的提示词可以：

- **提高准确性**：减少AI的幻觉和错误信息
- **控制输出**：精确指定格式、风格、长度
- **激发能力**：引导模型展现特定技能
- **提升效率**：减少反复修改的迭代次数

## 提示词的基本结构

一个高质量的提示词通常包含以下四个要素：

### 1. 角色设定 (Role)

告诉AI它应该扮演什么角色：

``````
你是一位经验丰富的软件架构师，擅长分布式系统设计，
有10年以上的互联网大厂工作经验。
``````

<InfoCard type="tip" title="角色设定技巧">

角色越具体，输出越专业。不要只说"你是专家"，而要说明**什么领域的专家**、**有什么经验**。

</InfoCard>

### 2. 任务描述 (Task)

清晰描述要完成的任务：

``````
请帮我设计一个电商平台的订单系统架构，
需要支持每秒10万笔订单的处理能力。
``````

### 3. 约束条件 (Constraints)

设定明确的限制条件：

```
要求：
- 使用微服务架构
- 数据库需要考虑分库分表
- 需要保证数据一致性
- 输出包含架构图说明和关键技术选型
```

### 4. 示例 (Examples)

提供示例帮助模型理解期望的输出格式：

```
例如：
输入: 用户下单购买手机
处理流程:
  1. 创建订单记录(状态: 待支付)
  2. 扣减库存
  3. 生成支付链接
输出: 订单号 + 支付链接
```

<InfoCard type="example" title="完整示例">

```
你是一位资深Python开发者。

请实现一个快速排序算法，要求：

- 使用TypeScript
- 包含完整的类型注解
- 添加详细的注释说明每步逻辑
- 时间复杂度为O(n log n)

例如：
输入: [5, 3, 8, 1, 9, 2]
输出: [1, 2, 3, 5, 8, 9]
```

</InfoCard>

## 常用技巧

### 1. Chain of Thought (思维链)

让模型逐步展示思考过程，大幅提高复杂问题的准确率：

```
请一步步分析这个问题：

1. 首先理解问题的核心要求
2. 列出可能的解决方案
3. 分析每个方案的优缺点
4. 选择最优方案并说明理由
5. 给出完整的实现步骤
```

<InfoCard type="tip" title="思维链的威力">

研究表明，对于复杂的数学和逻辑问题，添加"让我们一步步思考"(Let's think step by step)可以将准确率从18%提升到79%！

</InfoCard>

### 2. Few-Shot Learning (少样本学习)

通过示例让模型"举一反三"：

```
将以下英文翻译成中文，保持专业术语准确：

Input: The API endpoint returns a 401 Unauthorized response.
Output: 该API端点返回401未授权响应。

Input: Please implement rate limiting to prevent abuse.
Output: 请实现速率限制以防止滥用。

Input: The database connection pool is exhausted.
Output:
```

### 3. Role Playing (角色扮演)

```
你是一个严格的技术面试官。请对我回答的以下问题进行评分（1-10分），
并指出回答中的不足之处：

问题：请解释RESTful API的设计原则
我的回答：REST是一种架构风格...
```

### 4. Delimiter Technique (分隔符技术)

使用特殊标记分隔不同部分，避免混淆：

```
请分析以下代码的安全问题：

<code>
def authenticate_user(username, password): query = f"SELECT * FROM users WHERE
username='{username}' AND password='{password}'" return db.execute(query)
</code>

<task>
1. 识别安全漏洞
2. 说明漏洞原理
3. 提供修复方案
</task>
```

## 高级技巧

### 1. Output Formatting (输出格式化)

精确控制输出的格式：

```
请以以下JSON格式返回结果：

{
"title": "文章标题",
"summary": "100字以内的摘要",
"key_points": ["要点1", "要点2", "要点3"],
"tags": ["标签1", "标签2"],
"read_time": "预估阅读时间（分钟）"
}

文章内容：[在此输入文章]
```

### 2. Temperature Parameter (温度参数)

控制输出的"创造性"程度：

| 温度值    | 适用场景           | 特点               |
| --------- | ------------------ | ------------------ |
| 0.1 - 0.3 | 代码生成、事实问答 | 输出确定、可预测   |
| 0.5 - 0.7 | 一般对话、内容创作 | 平衡创造性和准确性 |
| 0.8 - 1.0 | 创意写作、头脑风暴 | 输出多样、有创意   |

<InfoCard type="warning" title="温度不是万能药">

温度参数只是影响输出的因素之一。如果提示词本身质量差，再好的温度参数也救不回来。**先把提示词写好，再调温度**。

</InfoCard>

### 3. Self-Correction Prompting (自我修正)

让AI自己检查和改进输出：

``````
请先写出你的答案，然后：

1. 检查答案是否有逻辑错误
2. 检查是否有遗漏的重要信息
3. 检查格式是否符合要求
4. 如果有问题，请修正后重新输出
   ``````

### 4. System Prompt (系统提示)

在API调用中使用system角色设定全局行为：

```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是一个专业的技术文档编写助手。你的回答应该：1. 准确 2. 简洁 3. 结构化 4. 包含代码示例"
    },
    {
      "role": "user",
      "content": "请解释什么是OAuth 2.0"
    }
  ]
}
```

## Prompt Framework 推荐

以下是一些经过验证的提示词框架：

### CREATE 框架

| 字母  | 含义               | 示例                     |
| ----- | ------------------ | ------------------------ |
| **C** | Character (角色)   | "你是一位资深数据科学家" |
| **R** | Request (请求)     | "请分析这份销售数据"     |
| **E** | Examples (示例)    | "类似这样的输出格式..."  |
| **A** | Adjustments (调整) | "用图表展示趋势"         |
| **T** | Type (类型)        | "以Markdown格式输出"     |
| **E** | Extras (额外)      | "重点标注异常数据"       |

### TAG 框架

- **T**ask (任务) - 要做什么
- **A**ction (行动) - 如何做
- **G**oal (目标) - 最终要达到什么

## 实践建议

<InfoCard type="tip" title="提示词工程师的日常">

1. **建立模板库**：将常用的提示词保存为模板
2. **持续迭代**：每次使用后记录效果，不断优化
3. **A/B测试**：对关键任务测试不同版本的提示词
4. **关注AI更新**：新模型可能有新的最佳实践

</InfoCard>

## 总结

提示词工程不是一门精确的科学，而是一门**需要不断实践和迭代**的艺术。

**核心原则**：

1. **具体明确** > 模糊宽泛
2. **提供上下文** > 直接提问
3. **给出示例** > 仅靠描述
4. **迭代优化** > 一次到位
